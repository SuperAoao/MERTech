Sigmoid

Softmax（可学习层权重的归一化）

在 function/model.py 的 HuggingfaceFrontend 里，当启用“加权融合隐藏层”（use_last=False）时，会对可学习的 layer_weights 做 softmax，把它们变成所有层的权重分布（每一层的权重和为 1），再用加权求和得到融合后的特征。

下面用一个“单个时间步 + 小维度”的例子把计算过程写出来。

图：从 hidden states 到加权融合

flowchart LR A[layer_weights logits
z0..zL] --> B[softmax
w0..wL] C[hidden_states
h0,h1,...,hL] --> D[加权求和
h = sum_i wi * hi] B --> D

输出例子（假设 3 层，hidden_dim=2）

假设在某个时间步 t：

  • 三层 hidden states：
    • h0 = [1, 2]
    • h1 = [3, 4]
    • h2 = [5, 6]
  • 可学习层权重 logits（layer_weights）：
    • z = [0.0, 1.0, 2.0]

softmax 后得到权重（近似）：

  • w = softmax(z) ≈ [0.0900, 0.2447, 0.6652]（三者和为 1）

融合后的输出： h = 0.0900*h0 + 0.2447*h1 + 0.6652*h2

计算（近似）：

  • 0.0900*[1,2] = [0.0900, 0.1800]
  • 0.2447*[3,4] = [0.7341, 0.9788]
  • 0.6652*[5,6] = [3.3260, 3.9912]

所以：

  • h ≈ [4.1501, 5.1500]

ReLU（分类头里的非线性）

在 function/model.py 的 Backend 里，分类头对投影特征先做 Dropout，再过 ReLU（x = self.relu(x)），引入非线性，让后续 Linear 输出 logits 时表达能力更强。

图：Dropout + ReLU + Linear 的数据流

flowchart TD X[proj(x) 得到特征
shape: [B,T,512]] --> D[Dropout(0.2)] D --> R[ReLU(max(0,·))] R --> C[classifier Linear → logits]

输出例子（ReLU 截断负值）

假设分类头在某个位置取到一个向量（ReLU 前）： u = [-1.2, 0.3, 2.0, -0.4]

ReLU 后： relu(u) = [0, 0.3, 2.0, 0]

然后把这个经过 ReLU 的激活送进 classifier = nn.Linear(512, class_size)， 得到该时间步、每个类别的 logits（再在训练/推理时用 BCEWithLogitsLoss 或 sigmoid 处理为 0~1 概率）。

Self-Supervised Learning, SSL

  1. 核心逻辑：屏蔽与预测 (Masked Language Modeling) MERT 借鉴了 BERT（自然语言处理领域）的思想。想象一下，你有一段古筝曲子：

掩码 (Masking)： 模型会随机把音频序列中的某些片段（比如 15% 的时间切片）给盖住（变成 0 或者噪音）。

预测 (Prediction)： 模型的任务是根据被盖住片段前后的声音，推断出中间被盖住的部分到底是什么。

怎么算拟合？ 如果模型能根据上下文“猜”出被盖住的频率分量或特征，说明它已经理解了音乐的统计规律（例如：听到“哆、来”之后，后面大概率跟着“咪”）。

  1. 具体的“拟合目标”：RVQ (残差矢量量化) 由于原始波形（Raw Waveform）包含太多随机噪声，直接预测波形很难。MERT 的做法是去预测离散化的编码（Codes）。

量化（Quantization）： 系统预先有一个“密码本”（Codebook），里面存了几千种常见的音频特征片段。

训练目标： 当一段音频被挡住时，模型输出一个特征向量（Tensor），让这个 Tensor 与那个“密码本”里对应的正确片段越接近越好。

拟合衡量： 使用 交叉熵损失 (Cross-Entropy Loss)。它不是在拟合“音高”，而是在拟合“这段音频在密码本里最像哪一个索引”。

  1. 多层次的特征提取 MERT 不仅仅拟合一种特征，它同时拟合多个层级的特征：

浅层： 拟合频谱特征（类似于音色、频率）。

深层： 拟合语义特征（类似于曲式、风格）。

这种“多任务并行”的方式，让它输出的 1024 维 Tensor 包含了极其丰富的信息。

Metrics

Precision / Recall / F1（公式）

对二分类（0/1）预测，定义：

  • (TP)（true positive）：预测为 1 且 GT 为 1
  • (TN)（true negative）：预测为 0 且 GT 为 0
  • (FP)（false positive）：预测为 1 但 GT 为 0
  • (FN)（false negative）：预测为 0 但 GT 为 1

则：

$$ Precision = \frac{TP}{TP + FP} $$

$$ Recall = \frac{TP}{TP + FN} $$

$$ F1 = \frac{2 \cdot Precision \cdot Recall}{Precision + Recall} = \frac{2TP}{2TP + FP + FN} $$

$$ Accuracy = \frac{TP + TN}{TP + TN + FP + FN} $$

micro-F1 / macro-F1（公式）

本项目里（以 IPT_label 为例）标签是一个二维 0/1 矩阵：(NUM_LABELS=7, T)；预测 pred_IPT 同形状。每个技法类别相当于一个“二分类任务”，在所有时间帧上统计 TP/FP/FN。

micro-F1（先全局加总再算）

把所有类别的 TP/FP/FN 先加总：

$$ TP_{\text{micro}}=\sum_{k=1}^{K} TP_k,\quad FP_{\text{micro}}=\sum_{k=1}^{K} FP_k,\quad FN_{\text{micro}}=\sum_{k=1}^{K} FN_k $$

然后：

$$ F1_{\text{micro}} = \frac{2TP_{\text{micro}}}{2TP_{\text{micro}} + FP_{\text{micro}} + FN_{\text{micro}}} $$

macro-F1（每类先算 F1，再平均）

$$ F1_{\text{macro}} = \frac{1}{K}\sum_{k=1}^{K} F1_k $$

其中 (K=7)（古筝 IPT 类别数）。

这些指标各自“侧重”什么

  • Precision（少误报）：更关心预测为 1 的里面有多少是真的（惩罚 FP）。适合“误报代价高”的场景（例如把不存在的技法/音符报出来很麻烦）。
  • Recall（少漏报）：更关心 GT 为 1 的里面抓到了多少（惩罚 FN）。适合“漏检代价高”的场景（例如漏掉一个真实 onset/技法会影响后续解析）。
  • F1（Precision 与 Recall 折中）：当你同时在意误报和漏报，用一个数平衡两者；比 Accuracy 更适合正负极不均衡（尤其是正例稀疏）的帧级任务。
  • Accuracy（总体正确率）：看所有位置里预测对的比例（TP+TN）。在本项目这种“绝大多数帧都是 0”的任务里，TN 会非常多，Accuracy 可能偏高但不代表模型真的会检测技法/音符（因此常作为辅助指标）。
  • micro-F1（大类/高频样本权重更大）：先把所有类的 TP/FP/FN 汇总再算，等价于按“出现次数/帧数”加权；常反映整体平均表现。
  • macro-F1（小类更公平）：每个类别先算 F1 再平均，每一类权重相同；能更敏感地暴露“某些稀有技法几乎不会”的问题。

假设某个 5 帧片段（t0–t4），GT（target_IPT）与预测（pred_IPT，阈值化后 0/1）如下（只画 3 类，其余 4 类全 0）：

GT（target_IPT）：

类别 \ 帧 t0 t1 t2 t3 t4
chanyin(0) 0 0 1 1 1
boxian(1) 0 0 0 0 0
shanghua(2) 0 1 1 0 0

Pred（pred_IPT）：

类别 \ 帧 t0 t1 t2 t3 t4
chanyin(0) 0 0 1 0 1
boxian(1) 0 1 0 0 0
shanghua(2) 0 1 0 0 0

逐类 TP/FP/FN 与 F1

  • chanyin(0)：

    • TP：t2,t4 → 2
    • FP：无 → 0
    • FN：t3（GT=1 但 Pred=0）→ 1
    • (F1_0=\frac{2TP}{2TP+FP+FN}=\frac{4}{4+0+1}=0.8)

  • boxian(1)：

    • GT 全 0，但 Pred 在 t1=1
    • TP=0, FP=1, FN=0
    • (F1_1=0)

  • shanghua(2)：

    • TP：t1 → 1
    • FP：无 → 0
    • FN：t2 → 1
    • (F1_2=\frac{2}{2+0+1}=0.667)

其余 4 类：GT 与 Pred 都全 0（常见实现会把它们的 F1 记为 0，或在 macro 时跳过“无正例类”，具体取决于指标函数实现）。

micro-F1（按所有类别/所有帧汇总）

把 7 类都算在内时（这里先只展示上面 3 类的汇总）：

  • (TP_{\text{micro}} = 2 + 0 + 1 = 3)
  • (FP_{\text{micro}} = 0 + 1 + 0 = 1)
  • (FN_{\text{micro}} = 1 + 0 + 1 = 2)

$$ F1_{\text{micro}}=\frac{2\cdot 3}{2\cdot 3 + 1 + 2}=\frac{6}{9}=0.667 $$

macro-F1（7 类平均）

若把其余 4 类当作 F1=0，则：

$$ F1_{\text{macro}}=\frac{F1_0+F1_1+F1_2+0+0+0+0}{7} =\frac{0.8+0+0.667}{7}=0.210 $$

（注意：macro 的数值会强烈依赖“空类别是否计入平均”的约定；你脚本里是直接把 7 个类别的 F1 相加再除以 7，所以属于“空类也计入”的 macro。）

Loss Functions（损失函数）

训练时要把模型输出的 logits 与标签做对齐，损失函数（loss）用于衡量“预测离真值有多远”，并指导反向传播更新参数。

常见损失函数与适用场景

  1. 交叉熵损失 CrossEntropyLoss（多分类 / 单标签）

    • 适用：每个样本/时间帧只属于一个类别（互斥 single-label）
    • 常见输出：softmax 概率或 logits

  2. 二元交叉熵 BCE / BCEWithLogitsLoss（多标签 / 二分类）

    • 适用：允许同一时间帧同时出现多个类别（multi-label），每个类别是一组独立的二分类任务
    • 常见输出：logits（未 sigmoid）
    • （BCEWithLogitsLoss 会把 sigmoid + BCE 合并，数值更稳定）

  3. 均方误差 MSELoss（回归）

    • 适用：连续数值回归任务（例如预测音高/参数的连续值）

  4. Huber / 平滑 L1（回归，抗离群点）

    • 适用：回归且希望对异常值更鲁棒

  5. Focal Loss（类别极度不均衡时）

    • 适用：正负样本极不均衡，希望更多关注难样本
    • 也是二分类/多标签任务常用的扩展

本项目为什么用 BCE（对应当前任务）

本项目的输出是 帧级 0/1 标签矩阵，例如：

  • IPT_label：(NUM_LABELS=7, T)，每个 IPT 类别在每帧上是独立的二分类（0 或 1）
  • pitch_label：(N_pitch=52, T)，同样是每个音高 bin 的独立二分类
  • onset_label：(1, T)，起音也是二分类（是否发生）

这种“每个类别是一组独立二分类”的结构，天然对应 BCE / BCEWithLogitsLoss：

  • 输入：模型输出 logits（不需要你在 forward 里做 sigmoid）
  • 标签：0/1
  • 输出：每个类别/每个帧的误差，loss 会对这些误差求和/平均

对应到代码（古筝任务）：

  • function/lib.py 里的 sp_loss() 使用了 nn.BCEWithLogitsLoss(...)

同时 onset 分支还做了正样本加权（让 onset 更“重要”）：

  • onset_loss() 里对正样本权重乘了常数（例如 10 * fl_target）

因此：BCE 是因为任务本质是 multi-label 的帧级检测（每帧、多类别都可能为 1），而不是互斥分类。

Model Design

本项目模型实现：function/model.py，核心是 MERT 特征提取器 + 多任务头（IPT / pitch / onset）+ onset 引导的注意力校正。

从宏观上理解：nn.Module 里的 forward() 是做什么的

在 PyTorch 里，forward() 定义了“输入张量如何一步步变成输出张量”，也就是模型的计算图。

  • 你平时写的 y = model(x)，实际调用的是 nn.Module.__call__()：
    • 它会处理 hooks、自动混合精度/一些内部逻辑
    • 然后再去调用你写的 forward(x)
  • 训练时（有梯度）：
    • forward() 产生输出与 loss
    • loss.backward() 会沿着 forward() 里记录的计算图反传梯度
  • 推理时（model.eval() + torch.no_grad()）：
    • 同样跑 forward() 得到输出
    • 但不构建梯度图，省显存、速度更快

所以：你把“网络结构 + 张量维度变换 + 多分支逻辑”都写在 forward() 里。

符号与本项目默认数值（方便对照 shape）

  • B：batch size（训练时 BATCH_SIZE=10，测试脚本常为 1）
  • L：输入波形长度（samples）
  • sr：输入给 MERT 的采样率 MERT_SAMPLE_RATE（默认 24000）
  • TIME_LENGTH：分块秒数（默认 5 秒）
  • T：帧数（time steps），默认 (T = FEATURE_RATE \times TIME_LENGTH = 75 \times 5 = 375)
  • D：MERT hidden dim（MERT-v1-95M 对应 768；330M 对应 1024）
  • K：IPT 类别数（NUM_LABELS=7）
  • P：pitch bin 数（MAX_MIDI-MIN_MIDI+1 = 87-36+1 = 52）
  • class_num：联合输出维度，代码里设为 K * P = 364

当输入是一个 5 秒 chunk 时，波形长度大约：

$$ L = sr \times TIME_LENGTH = 24000 \times 5 = 120000 $$

SSLNet：逐层张量 shape（按 forward 顺序）

下面的 shape 来自 SSLNet.forward() 的真实计算路径（function/model.py），并用本项目变量解释每一步。

Step 0：输入

  • 输入波形 x（来自 Wav2Vec2FeatureExtractor 的 input_values）：
    • 常见 shape：[B, 1, L] 或 [B, L]
    • 代码第一行做了：x = x.squeeze(dim=1)，因此进入 frontend 前变成：
      • [B, L]（示例：[10, 120000] 或 [1, 120000]）

Step 1：HuggingfaceFrontend（MERT）

  • 输出隐藏表示 h = frontend(x)：
    • shape：[B, T_mert, D]
    • 其中 T_mert 是 MERT 内部下采样后的帧数（由模型结构决定，通常与输入长度成比例）
    • 示例：[B, T_mert, 768]

我是如何判断 MERT 输出结构/shape 的

这部分依据不是“猜”，而是来自 HuggingFace AutoModel 的返回约定 + 你代码对返回值的访问方式：

  • 为什么能确定有 last_hidden_state / hidden_states

    • 你的代码显式写了 x = self.model(x, output_hidden_states=True)，并随后访问：
      • x["last_hidden_state"]
      • x["hidden_states"]
    • 这说明 MERT 的返回对象是 HuggingFace 标准的 ModelOutput（可用 dict key 取值），并且启用了 hidden_states。

  • 为什么能确定是三维 [B, T, D]

    • HuggingFace 的 encoder 类模型约定 last_hidden_state 为 (batch, time, hidden_size)。
    • 你的后端 Backend 直接接收 frontend 输出并喂给 nn.Linear(self.feature_dim, ...)，因此最后一维必须是 hidden size。

  • 为什么能确定 D=768（或 1024）

    • Backend.__init__() 把 feature_dim 写死为：
      • 768（encoder_size=12，对应 95M）
      • 1024（encoder_size=24，对应 330M）
    • 如果 frontend 输出最后一维不是 768/1024，Linear 会维度不匹配直接报错；所以这里是“由后端约束反推”出来的。

  • 为什么时间维只写 T_mert，不写死为 375

    • T=375 来自标签帧率 FEATURE_RATE=75 与 TIME_LENGTH=5，这是 标签时间分辨率。
    • T_mert 来自 MERT 内部下采样后的时间分辨率，两者不一定相等。
    • 你测试脚本里也专门做了 f_pred[:, :target.shape[-1]] 裁切对齐，说明模型输出时间长度与标签时间长度可能存在差异。

说明：你的标签帧率是 FEATURE_RATE=75（即 T=375/5 秒），而 T_mert 是 MERT 的时间分辨率；两者不一定相等，因此测试脚本里会用 f_pred[:, :target.shape[-1]] 做裁切对齐。

Step 2：Backend（联合输出：IPT×pitch）

  • out, feature = backend(h)
    • backend 的最后一层是 Linear(..., class_num)，因此：
    • out shape：[B, T_mert, class_num]
    • 本项目 class_num = K*P = 364
    • 示例：[B, T_mert, 364]

Step 3：reshape 成网格（K×P）

  • out = out.view(B, T_mert, K, P)
    • shape：[B, T_mert, 7, 52]

Step 4：边缘化得到 IPT / pitch logits

  • IPT logits（对 pitch 维求和）：

    • IPT_pred = sum(out, dim=3)
    • shape：[B, T_mert, K] → [B, T_mert, 7]

  • pitch logits（对 IPT 维求和）：

    • pitch_pred = sum(out, dim=2)
    • shape：[B, T_mert, P] → [B, T_mert, 52]

Step 5：onset 分支 logits

  • onset_pred, _ = backend_onset(h)
    • backend_onset 的输出维度是 1
    • shape：[B, T_mert, 1]

Step 6：onset 引导的注意力校正（Attnet）

代码里先 stop-gradient，再拼接：

  • onset_pred_deta = onset_pred.detach()（shape 不变）
  • IPT_onset_cat = cat(IPT_pred, onset_pred_deta, dim=2)
    • shape：[B, T_mert, K+1] → [B, T_mert, 8]
  • pitch_onset_cat = cat(pitch_pred, onset_pred_deta, dim=2)
    • shape：[B, T_mert, P+1] → [B, T_mert, 53]

然后通过 Backend_Attnet（内部是 MultiheadAttention(batch_first=True) + Linear）：

  • backend_attnet_IPT(IPT_onset_cat) 输出：
    • shape：[B, T_mert, K] → [B, T_mert, 7]
  • backend_attnet_pitch(pitch_onset_cat) 输出：
    • shape：[B, T_mert, P] → [B, T_mert, 52]

Step 7：最终输出布局（转成 [B, C, T]）

模型最后转置时间维到最后，便于和标签 (C, T) 对齐：

  • IPT_pred_out = ... .transpose(-1, -2)
    • shape：[B, K, T_mert] → [B, 7, T_mert]
  • pitch_pred_out = ... .transpose(-1, -2)
    • shape：[B, P, T_mert] → [B, 52, T_mert]
  • onset_pred_out = onset_pred.transpose(-1, -2)
    • shape：[B, 1, T_mert]

推理阶段为什么还要 sigmoid（与 shape 的关系）

模型输出的是 logits，不是概率；评测脚本对每一路都做：

  • squeeze(0)：batch=1 时从 [1, C, T] 变成 [C, T]
  • sigmoid：把 logits 映射到 [0,1]
  • [:, :target.shape[-1]]：裁到和标签的时间长度一致（标签是 (C, T_gt)）

这对应你在 test_train_frame_ao 里的处理逻辑。

function/model.py 各个类的 forward() 分别做什么

下面按源码中的类逐个说明（输入 → 处理 → 输出）。

SSLNet.forward(x)

  • 输入：波形 x（常见 [B, 1, L] 或 [B, L]）
  • 处理：
    • squeeze(dim=1)：去掉多余的 channel 维（若存在）→ [B, L]
    • frontend(x)：MERT 提取序列特征 → [B, T_mert, D]
    • backend(h)：输出联合 logits → [B, T_mert, class_num]
    • view(..., K, P)：重排成技法×音高网格 → [B, T_mert, 7, 52]
    • sum(dim=3) 得 IPT logits → [B, T_mert, 7]
    • sum(dim=2) 得 pitch logits → [B, T_mert, 52]
    • backend_onset(h) 得 onset logits → [B, T_mert, 1]
    • detach()：切断 onset 梯度，作为条件特征拼接进其它分支
    • Backend_Attnet：自注意力校正 IPT/pitch → [B, T_mert, 7] / [B, T_mert, 52]
    • transpose(-1,-2)：把时间维换到最后 → [B, C, T_mert]
  • 输出（logits，未 sigmoid）：
    • IPT：[B, 7, T_mert]
    • pitch：[B, 52, T_mert]
    • onset：[B, 1, T_mert]

HuggingfaceFrontend.forward(x)

  • 输入：波形 [B, L]
  • 处理：
    • AutoModel(..., output_hidden_states=True)：拿到最后一层与所有层 hidden states
    • use_last=True：用 last_hidden_state（[B, T_mert, D]）
    • use_last=False：stack 所有层再用可学习 layer_weights 做层加权融合（权重经 softmax 归一化）
    • F.pad(..., mode='reflect')：在时间维做 1 帧反射 padding（边界补齐）
  • 输出：序列特征 h，形状大致 [B, T_mert, D]

Backend.forward(x)

  • 输入：序列特征 [B, T, D]
  • 处理：
    • Linear(D→512) 投影
    • 可选 mean（若 frame=False，变 clip-level）
    • Dropout → ReLU → Linear(512→class_size) 输出 logits
  • 输出：
    • logits：帧级时 [B, T, class_size]
    • feature：投影后的中间特征（同维度，便于额外使用）

self_attn.forward(x)

  • 输入：[B, T, C]
  • 处理：
    • MultiheadAttention(x,x,x) 自注意力
    • 残差：att(x)+x
  • 输出：[B, T, C]

Backend_Attnet.forward(x)

  • 输入：
    • IPT 路：[B, T, 8]（7 个 IPT + 1 个 onset）
    • pitch 路：[B, T, 53]（52 个 pitch + 1 个 onset）
  • 处理：
    • 先 self_attn 做时序注意力融合
    • 再 Linear(feature_dim→class_size) 投影回目标类别数
  • 输出：
    • IPT：[B, T, 7]
    • pitch：[B, T, 52]
